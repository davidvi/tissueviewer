# Plan: Histogram-Based Intensity Range Control for Immunofluorescence Viewer

## Context & Current State

The viewer already has significant groundwork in place:

- **`ChannelHistogram.vue`** — A fully implemented component that renders an SVG histogram of pixel distribution for a channel, with draggable `low`/`high` handles. It fetches a low-resolution overview tile from the server (rendered in grayscale "white" color at full 0–1 range) and bins the R-channel pixel values into 64 buckets. It emits `update:low`, `update:high`, and `change` events.
- **`slideView.vue`** — Already mounts `<channel-histogram>` for each active channel with `v-model:low` / `v-model:high` bound to `channel.low` / `channel.high`.
- **`slideContent.js` (`reloadSlide`)** — Already builds the DZI URL including `minString` and `maxString` path segments: `/{location}/{chs}/{rgb}/{colors}/{gains}/{mins}/{maxs}/{slideName}.dzi`.

**The client side is essentially complete.** The histogram UI is already wired up.

### The Gap: Server Does Not Apply min/max Range

The server routes in `server.py` only accept 6 path segments — they do **not** parse the `mins` and `maxs` values that the client is sending. The URL the client produces is:

```
/{location}/{chs}/{rgb}/{colors}/{gains}/{mins}/{maxs}/{file}.dzi
```

But the server routes are:

```python
@app.get("/{location}/{chs}/{rgb}/{colors}/{gains}/{file}.dzi")
@app.get("/{location}/{chs}/{rgb}/{colors}/{gains}/{file}_files/{level}/{loc_x}_{loc_y}.jpeg")
```

The `mins` and `maxs` segments are currently being absorbed into the `{file}` path parameter as `mins/maxs/slideName`, which causes incorrect file path resolution. This is a **bug** that currently breaks the min/max feature entirely.

Additionally, `connect.py`'s `get_combined_image()` does not apply any range remapping — it only applies a gain multiplier: `np.clip(image * intensity, 0, 255)`.

---

## Goal

Make the histogram-based lower/upper intensity range control fully functional end-to-end:
1. The server routes must parse `mins` and `maxs`.
2. The image renderer must remap each channel's pixel values so that `min` maps to 0 and `max` maps to 255 (i.e., stretch the [min, max] window to full 8-bit range), then apply the gain multiplier.

---

```python
def get_combined_image(self,
                       image_id=0,
                       dzi_zoom_level=1,
                       channels=[0],
                       intensities=[1.0],
                       colors=['blue'],
                       is_rgb=False,
                       tile_x=0,
                       tile_y=0,
                       mins=None,    # NEW: list of floats 0.0–1.0
                       maxs=None,    # NEW: list of floats 0.0–1.0
                       ) -> np.array:
```

**Range remapping logic** (applied before gain, per channel):
```python
min_val = (mins[i] if mins else 0.0) * 255.0
max_val = (maxs[i] if maxs else 1.0) * 255.0
range_val = max(max_val - min_val, 1e-6)

# Remap: pixels below min_val → 0, pixels above max_val → 255
remapped = np.clip((image.astype(np.float32) - min_val) / range_val * 255.0, 0, 255)

# Apply gain
enhanced = np.clip(remapped * intensity, 0, 255).astype(np.uint8)
```

This replaces the current single-line: `enhanced_image = np.clip(image * intensity, 0, 255).astype(np.uint8)`

## Design Decisions (confirmed with user)

1. **Histogram sampling**: Use a **server-side `/histogram` endpoint** that samples the full image for accurate representation (important for sparse tissue where a single corner tile would be misleading).
2. **Labels**: Keep **percentages 0–100%** (normalized, bit-depth independent).
3. **Gain slider**: Keep hidden — the `low`/`high` range handles brightness adjustment more intuitively for immunofluorescence.

## Additional Step: Server-Side Histogram Endpoint

### Step 3 — Add `/histogram` API endpoint in `server/server.py`

Add a GET endpoint that reads the lowest-resolution zoom level of a given channel, samples pixel values across the full image, and returns a 64-bin histogram as JSON:

```
GET /histogram/{location}/{channel}/{file}
→ { "bins": [count0, count1, ..., count63] }
```

**Implementation:**
- Open the zarr file
- Access pyramid level 0 (lowest resolution / smallest overview)
- Read the entire array for the specified channel
- Normalize pixel values to 0–1 range (divide by 255 since stored as uint8 after zarr conversion, or by max dtype value)
- Bin into 64 buckets
- Return as JSON

```python
@app.get("/histogram/{location}/{channel}/{file}")
async def get_histogram(location: str, channel: int, file: str):
    path_to_zarr = os.path.join(settings.SLIDE_DIR, location, file + ".zarr")
    ome_connection = OmeZarrConnector(path_to_zarr)
    bins = ome_connection.get_channel_histogram(channel_index=channel, num_bins=64)
    return JSONResponse(content={"bins": bins})
```

Add `get_channel_histogram()` to `OmeZarrConnector` in `connect.py`:

```python
def get_channel_histogram(self, channel_index: int, num_bins: int = 64) -> list:
    """
    Compute a histogram of pixel values for a channel using the lowest
    resolution pyramid level (full image coverage, fast).
    """
    root = self.zarr_connection[0]
    # Level 0 in the zarr is the lowest resolution (smallest)
    da_zarr = da.from_zarr(root[0])
    # Shape: [batch, channel, z, y, x]
    arr = da_zarr[0, channel_index, 0, :, :].compute()
    
    # Normalize to 0.0–1.0
    max_val = np.iinfo(arr.dtype).max if np.issubdtype(arr.dtype, np.integer) else 1.0
    flat = arr.flatten().astype(np.float32) / max_val
    
    counts, _ = np.histogram(flat, bins=num_bins, range=(0.0, 1.0))
    return counts.tolist()
```

### Step 4 — Update `ChannelHistogram.vue` to use the new endpoint

Replace the current DZI-fetch + canvas pixel-counting approach with a single call to the new histogram endpoint:

```js
async fetchHistogram() {
  if (!this.storeSampleName) return;
  this.loading = true;
  try {
    const url = `${baseUrl}/histogram/${this.storeLocation}/${this.channelNumber}/${this.storeSampleName}`;
    const resp = await axios.get(url);
    this.bins = resp.data.bins;
  } catch (e) {
    console.error('ChannelHistogram: fetch failed', e);
    this.bins = new Array(NUM_BINS).fill(1);
  } finally {
    this.loading = false;
  }
},
```

This is simpler, faster (one request vs. two), and covers the full image.
