# Plan: Overlay UI Improvements

## Goal
Three changes to the CSV point overlay UI in `client/src/views/slideView.vue`:

1. **Remove X/Y factor controls** — delete `overlayScaleX` and `overlayScaleY` data properties, their template inputs, and the `applyOverlayScale` method (plus remove the scale logic from `displayPointOverlays`).
2. **Add "Max points" input** — expose `maxVisiblePoints` (currently hardcoded at `250`, never shown in UI) as a user-editable numeric input in the overlay settings panel.
3. **Add per-category enable/disable toggles** — let users show/hide individual label categories in the overlay legend without unloading the CSV.

---

## Affected File

`client/src/views/slideView.vue` (single file, all changes are local component state)

---

## Change 1 — Remove X/Y Factor Controls

### Template (lines ~251–274)
Remove the entire `<div class="flex space-x-2 mb-2">` block that contains the X factor and Y factor `<input>` elements.

### Data (lines 363–364)
Remove:
```js
overlayScaleX: 1,
overlayScaleY: 1,
```

### Methods
- Delete `applyOverlayScale()` method entirely (lines ~884–890).
- In `displayPointOverlays(points)` (lines ~891–904): remove the scale logic. The method currently reads `scaleX`/`scaleY` and multiplies coordinates. Replace with a direct copy (no scaling):
  ```js
  displayPointOverlays(points) {

    this.clearPointOverlays();
    this.updateVisiblePoints();
  },
  ```

## Change 2 — Add "Max points" Input

`maxVisiblePoints` is already a `data` property (line 354, default `250`). We just need to expose it in the UI.

### Template — add after the overlay visibility toggle block (around line 321), still inside the `v-if="overlayFiles.length > 0"` section:
```html
<!-- Max visible points control -->
<div class="mt-2">
  <label class="text-white text-xs block">Max points</label>
  <input
    type="number"
    min="1"
    v-model.number="maxVisiblePoints"
    @change="updateVisiblePoints"
    class="block w-full mt-1 pl-2 pr-2 py-1 text-base border-gray-300 focus:outline-none focus:ring-primary focus:border-primary sm:text-sm rounded-md"
    placeholder="250"
  />
</div>
```

No logic changes needed — `updateVisiblePoints()` already reads `this.maxVisiblePoints` when sampling points.

## Change 3 — Per-Category Enable/Disable Toggles

### New data property
Add a `disabledOverlayCategories` set (stored as an array for reactivity) to `data()`:
```js
disabledOverlayCategories: [],   // array of label strings that are currently hidden
```

### HUD Legend — make each legend item interactive (lines 26–34)
Replace the static legend rows with interactive toggle rows that visually indicate enabled/disabled state:
```html
<div v-if="activeOverlayLegend">
  <h3 class="text-lg font-semibold mb-2 mt-3">Overlay Legend</h3>
  <div
    v-for="(color, label) in activeOverlayLegend"
    :key="label"
    class="mb-1 cursor-pointer"
    @click="toggleOverlayCategory(label)"
  >
    <div class="flex items-center space-x-2">
      <div
        class="w-4 h-4 rounded-full"
        :style="{
          backgroundColor: color,
          opacity: disabledOverlayCategories.includes(label) ? 0.3 : 1
        }"
      ></div>
      <p
        class="text-sm"
        :style="{ opacity: disabledOverlayCategories.includes(label) ? 0.4 : 1 }"
      >
        {{ label }}
      </p>
    </div>
  </div>
</div>
```

### New method `toggleOverlayCategory(label)`
```js
toggleOverlayCategory(label) {
  const idx = this.disabledOverlayCategories.indexOf(label);
  if (idx === -1) {
    this.disabledOverlayCategories.push(label);
  } else {
    this.disabledOverlayCategories.splice(idx, 1);
  }
  this.updateVisiblePoints();
},
```

### Update `updateVisiblePoints()` to filter by enabled categories
After computing `pointsInViewport`, add a filter step:
```js
const pointsInViewport = this.allPointOverlays.filter(point => {
  const inViewport = (
    point.location.x >= viewportBounds.x &&
    point.location.x <= viewportBounds.x + viewportBounds.width &&
    point.location.y >= viewportBounds.y &&
    point.location.y <= viewportBounds.y + viewportBounds.height
  );
  const categoryEnabled = !this.disabledOverlayCategories.includes(point.description);
  return inViewport && categoryEnabled;
});
```

### Reset `disabledOverlayCategories` when a new overlay file is loaded
In `displayPointOverlays()`, reset the disabled set so stale hidden categories from a previous file don't carry over:
```js
this.disabledOverlayCategories = [];
```
