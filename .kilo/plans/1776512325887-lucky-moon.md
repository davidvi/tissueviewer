# TissueViewer — pip-installable Python package conversion

## 1. Investigation summary

### Current state (what actually runs)

- The server that is used today is **`server/ome-server.py`** (OME-TIFF, pyramid tiles via `tifffile` + `zarr`).
- `server/server.py` is the **legacy** server: OME-Zarr only, requires `bioformats2raw` (Java) running inside Docker. It references `/bin/rm`, `/usr/bin/du`, `/tv-import`, `/tv-store`, and `OmeZarrConnector`.
- `server/watch_folder.py` watches `/tv-import` and shells out to bioformats2raw — Docker-only.
- The Vue client is already **pre-built** in `client/dist/` (index.html, `assets/`, `images/`, `favicon.ico`). The source in `client/src/` and `node_modules/` is only needed when rebuilding the UI.
- Client API surface actually used (grep of `client/src`):
  - `GET /samples.json?location=<loc>` → sample list
  - `GET /{location}/{chs}/{rgb}/{colors}/{gains}/{file}.dzi` and the windowed variant
  - `GET /{location}/{chs}/{rgb}/{colors}/{gains}/{file}_files/{level}/{x}_{y}.jpeg` and the windowed variant
  - `GET /histogram/{location}/{channel}/{file}`
  - `POST /save/{location}/{file}` (writes `{file}.sample.json`)
  - `POST /uploadSample`, `POST /sampleStats`, `POST /deleteSample` — **upload/delete view, to be dropped**
- Server expects files named `<name>.ome.tif(f)` inside `SLIDE_DIR/<location>/`; writes/reads `<name>.sample.json` and `<name>.metadata.json` (metadata cache) next to them.
- Hard-coded paths: `TV_SLIDE_DIR=/tv-store` (env-override), `/tmp`, `client_dir = ../client/dist` relative to server file.
- Docker-specific files: `Dockerfile`, `cloud-deploy/Dockerfile`, `cloud-deploy/README.md`, all `server/run_*.sh`.

### Dependency reality (verified against live envs)

The conda env literally named `tissueviewer` is empty (just pip/setuptools/wheel). **The server is confirmed working under the `ImmunoViewer` conda env (Python 3.11).** `requirements.txt` in the repo is stale and does not reflect what's actually installed in that env.

Actual imports in `ome-server.py`:
`pathlib, os, json, math, io.BytesIO, typing, threading, time, concurrent.futures` (stdlib) plus `cv2, numpy, tifffile, zarr, fastapi, uvicorn, xml.etree.ElementTree`.

`ome-server.py` already has a zarr-2/zarr-3 compatibility branch (checks `.members()` vs `.items()`), so we keep it version-flexible.

### User-confirmed scope

- OME-TIFF only for v1 (no Zarr dirs, no bioformats2raw, no Java).
- No upload / delete / sampleStats / watch-with-conversion endpoints.
- `--watch` becomes a lightweight filesystem rescan on change (no conversion).
- Location model: synthetic `public` root; immediate subdirs become locations; `--recursive` flattens nested dirs into slash-joined locations. No client changes.
- Package: `src/` layout + `pyproject.toml` (setuptools backend).
- Client shipped as `client/dist/` copied into `src/tissueviewer/web/` at build time via `package-data`.
- Dependency strategy: loose lower bounds with safety caps.
- Python 3.10+.
- Legacy/Docker files **deleted** entirely (git history preserves them).

## 4. Code refactor plan (module by module)

### `src/tissueviewer/__init__.py`
```python
__version__ = "0.2.0"
from .app import create_app
from .config import Config
__all__ = ["__version__", "create_app", "Config"]
```

### `src/tissueviewer/config.py`

- Plain `@dataclass` `Config` (drops pydantic-settings — unused in the working code path):
  - `data_dir: Path` (where the slides live; was `SLIDE_DIR`)
  - `host: str = "127.0.0.1"`
  - `port: int = 8000`
  - `save_enabled: bool = True`
  - `recursive: bool = False`
  - `single_file: Optional[Path] = None`
  - `colors: list[str] = [...default 7 names...]`
  - `tile_cache_size: int = 4`
  - `log_level: str = "INFO"`
- `Config.from_sources(cli_args, config_path)` merges in this precedence: **CLI args > env vars (`TV_*`) > YAML file > defaults**.
- YAML parser uses `yaml.safe_load`. Env vars keep the existing `TV_` prefix for backward compat: `TV_SLIDE_DIR`, `TV_HOST`, `TV_PORT`, `TV_SAVE`.

### `src/tissueviewer/cli.py`

`argparse` with this surface:

```
usage: tissueviewer [TARGET] [options]

TARGET (optional):
  path to a file or directory. If omitted, --config must be provided
  or TV_SLIDE_DIR must be set.

options:
  -r, --recursive            include nested subdirectories
  -c, --config FILE          YAML config file
  -p, --port N               listen port (default 8000)
  -H, --host HOST            bind address (default 127.0.0.1)
  -o, --open                 open browser after startup
  -w, --watch                rescan target on filesystem changes
      --validate             check file/dir compatibility and exit
      --no-save              disable writing .sample.json files
      --log-level LEVEL      DEBUG | INFO | WARNING | ERROR
  -V, --version              show version and exit
  -h, --help                 show help and exit
```

CLI behaviour:

1. Parse args; load YAML if `--config`; build final `Config`.
2. Mode detection in `cli.py`:
   - `TARGET` is a file ending in `.ome.tif`/`.ome.tiff` → single-file mode. We stage a virtual directory in memory: one entry under location `public` pointing at the real file via an in-memory mapping held by the app state.
   - `TARGET` is a directory → directory mode. `recursive` flag controls depth.
   - No `TARGET` and no `data_dir` in config → print help + error.
3. If `--validate`: run `validation.run_validation(config)` and exit 0/1.
4. Otherwise call `server.run_server(config)` which:
   - Builds `app = create_app(config)`
   - Prints the URL
   - If `--open`, schedules `browser.open_after_delay(url, 1.0)` on a background thread
   - Calls `uvicorn.run(app, host=..., port=..., log_level=...)` with **reload disabled** (reload requires an import string and is dev-only; drop the current `reload=True`).

### `src/tissueviewer/logging_setup.py`

Thin wrapper: configures root logger + uvicorn access logger with a concise formatter. Replaces the ad-hoc `print()` calls in `ome-server.py`. All modules use `logger = logging.getLogger(__name__)`.

### `src/tissueviewer/discovery.py`

Two responsibilities:

1. **Location mapping** (virtual layout presented to the client):
   - Single-file mode → `{ "public": [<that file>] }`
   - Directory mode, non-recursive → root of dir maps to `public`; each immediate subdir maps to its own location named after the subdir
   - Directory mode, recursive → walk full tree; files directly in root go to `public`; files in subdirs go to the slash-joined relative path of their parent (e.g. `2024/batch_A`). Slash-safe because the client URL-encodes it.
2. **File listing & metadata** — lift `find_ome_tiffs`, `get_metadata_for_file`, the metadata-cache helpers, and `_process_single_file` out of `ome-server.py`. Operate on the location-specific directory returned from the map. Single-file mode skips directory scanning and returns a one-element list.

The app state holds:
```
@dataclass
class AppState:
    config: Config
    tile_cache: TiffCache
    location_map: dict[str, Path]   # location -> absolute dir or parent dir
    single_file_path: Path | None   # non-None only in single-file mode
```

Routes resolve a requested file path via `location_map[location] / f"{file}.ome.tif(f)"` (with the single-file short-circuit).

### `src/tissueviewer/formats/ome_tiff.py`

- Move `OmeTiffPyramid`, `TiffCache`, `extract_channel_info`, and the color helpers out of `ome-server.py` verbatim (logic already works) but:
  - Replace bare `print` with `logger.debug`/`logger.warning`.
  - Remove the hard-coded `/tmp` and `settings.SLIDE_DIR` references.
  - Accept an explicit `tile_size` from `Config`.

### `src/tissueviewer/formats/base.py` + `__init__.py`

Minimal extension point for future formats:

```python
class ImageHandler(Protocol):
    def open(self, path: Path) -> "PyramidLike": ...
def get_handler(path: Path) -> ImageHandler: ...  # dispatches by suffix
```

Today only the OME-TIFF handler is registered; the registry exists so adding plain TIFF / OME-Zarr later is a single new module.

### `src/tissueviewer/routes/*`

Split `ome-server.py` endpoints one-per-file:

- `client.py` — serves index / favicon / assets / images from `importlib.resources.files("tissueviewer") / "web"`. Uses `StaticFiles(directory=...)` with a `Path` obtained via `importlib.resources.as_file` so it works both from source checkout and installed wheel.
- `samples.py` — `GET /samples.json` — uses `AppState.location_map` and `discovery.list_samples(state, location)`.
- `tiles.py` — both `get_dzi` and `get_tile` plus their windowed variants. Resolves `.ome.tif(f)` via `state.resolve_file(location, file)`.
- `histogram.py` — unchanged logic, new import path.
- `save.py` — unchanged logic except `slide_dir` comes from `state`.

### `src/tissueviewer/app.py`

```python
def create_app(config: Config) -> FastAPI:
    state = build_state(config)         # discovery, cache, location_map
    app = FastAPI(title="TissueViewer", version=__version__)
    app.state.tv = state
    app.add_middleware(CORSMiddleware, allow_origins=["*"], ...)
    register_routes(app, state)
    return app
```

### `src/tissueviewer/server.py`

```python
def run_server(config: Config) -> None:
    app = create_app(config)
    if config.open_browser:
        threading.Thread(target=browser.open_when_ready, args=(url,), daemon=True).start()
    if config.watch:
        state.start_watcher()   # schedules discovery.refresh() every N seconds
    uvicorn.run(app, host=config.host, port=config.port, log_level=config.log_level.lower())
```

### `src/tissueviewer/watcher.py`

Stdlib-only (`threading` + `time`) mtime-poll loop that invalidates the `location_map` sample cache and closes+re-opens pyramids whose file mtime changed. No new dependency like `watchdog` (keeps wheel slim). Sample list is re-derived lazily on next `/samples.json` request; watcher just bumps a generation counter.

### `src/tissueviewer/validation.py`

`run_validation(config)`:
- Walk discovered files
- For each, try `tifffile.TiffFile(path).series[0]` and report axes, shape, dtype, channel count
- Report pass/fail counts
- Exit non-zero if any failures

### `src/tissueviewer/browser.py`

`open_when_ready(url, max_wait=5.0)`: poll with `socket.create_connection(host, port)`; once succeeds, `webbrowser.open(url)`.

## 8. Testing plan

Smoke-level pytest (tests run on every commit, no network):

1. **`test_cli.py`**: argparse parses each flag correctly; unknown target path → error; `--version` prints and exits; `--help` exits 0.
2. **`test_config.py`**: precedence CLI > env > YAML > defaults; YAML round-trip.
3. **`test_discovery.py`**:
   - Create a temp tree `a.ome.tif`, `sub/b.ome.tif`, `sub/nested/c.ome.tif`.
   - Single file → `{"public": [a]}`.
   - Non-recursive dir → `{"public": [a], "sub": [b]}` (c hidden).
   - Recursive dir → `{"public": [a], "sub": [b], "sub/nested": [c]}`.
4. **`test_app_smoke.py`** (uses `httpx.AsyncClient` + FastAPI `TestClient`):
   - `GET /` returns HTML containing `<div id="app">`.
   - `GET /favicon.ico` returns binary.
   - `GET /samples.json?location=public` returns `{"samples": [...], "save": true, "colors": [...]}`.
   - `POST /save/public/{name}` with `--no-save` returns 403.
   - Tile/DZI endpoints covered against a **synthesized tiny OME-TIFF** written with `tifffile.imwrite(..., ome=True, metadata={...})` (a fixture creates a 256×256, 2-channel float32 file on the fly — no large binary fixtures checked in).
5. **`test_validation.py`**: valid file passes; corrupted file reports failure; missing path exits 1.

Manual verification checklist (documented in CONTRIBUTING):

- `pip install .` into fresh venv; `tissueviewer --version` works.
- `tissueviewer <single .ome.tiff>` → visit `http://127.0.0.1:8000/`, see the slide under `public`.
- `tissueviewer <folder>` with 2 subfolders of slides → locations appear in the dropdown.
- `--recursive` discovers nested files.
- `--port 9000 --host 0.0.0.0 --open` binds and opens browser.
- `--watch` picks up a new file without restart.
- `--validate` on a directory reports each file's status and exits non-zero on a broken file.

## 10. Open items / risks

- **`opencv-python-headless` swap**: safe given only `resize/cvtColor/imencode` are used, but worth a single-run visual spot check of a generated tile to confirm pixel output is identical.
- **zarr 2 vs 3**: we keep the existing compat branch; pinned `zarr<4` guards against a future breaking release.
- **imagecodecs wheels on Apple Silicon**: confirmed wheels exist on PyPI for py310/311/312 on macOS arm64 — no source build required.
- **Reload mode**: intentionally dropped; `--reload` was dev-only and relied on an import-string (`ome-server:app`) that the new module path would break. Developers can use `uvicorn tissueviewer.app:create_app --factory --reload` manually.
- **`feature-overlay/`**: kept in the repo as research artifacts, explicitly excluded from the wheel via `MANIFEST.in`. Confirm with user before deletion if a cleaner repo is preferred — default is keep.
- **Legacy HTTP endpoints in client**: `/uploadSample`, `/sampleStats`, `/deleteSample` still get called by the Vue `uploadView`. With those endpoints returning 404 the upload page will simply error out; documented in MIGRATION. A follow-up PR can hide the upload route from the Vue router, but that requires a client rebuild and is out of scope for v1.
